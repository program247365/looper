## After PR Merge

### 1. Switch to main and pull
```bash
git checkout main
git pull origin main
```

### 2. Verify version
```bash
grep "^version" Cargo.toml
# Should show: version = "0.3.3"
```

### 3. Build and test
```bash
make build-release
make test

# Optional: Run benchmarks to verify performance
make bench-playback
make bench-analyze
```

### 4. Create release (automated)
```bash
make release
```

This will:
- ✅ Tag v0.3.3
- ✅ Push tag to GitHub
- ✅ Create GitHub release with auto-generated notes
- ✅ Update Homebrew formula with new version and SHA256
- ✅ Push Homebrew formula to tap repository

### 5. Verify release

**GitHub Release:**
- Visit: https://github.com/program247365/looper/releases
- Verify v0.3.3 is published
- Check release notes

**Homebrew Formula:**
- Visit: https://github.com/program247365/homebrew-tap
- Verify Formula/looper.rb updated to v0.3.3
- Check SHA256 hash is correct

### 6. Test installation
```bash
# Update local Homebrew
brew update

# Upgrade looper
brew upgrade program247365/tap/looper

# Verify version
looper --version  # Should show v0.3.3 (when version flag is added)

# Test that it works
looper play --url tests/fixtures/sound.mp3
```

### 7. Monitor CPU usage
```bash
# In another terminal
ps -o %cpu,rss -p $(pgrep looper)

# Expected: ~15-20% CPU (was 66.5%)
```

## Release Notes Template

Copy this for the GitHub release notes (auto-generated by gh, but can be edited):

```markdown
# v0.3.3 - Performance Release 🚀

## Major Improvement

**50-70% CPU reduction** through TUI render optimization!

### Before
- CPU: 66.5% during playback
- Rendering 33 FPS constantly (even when paused)
- Excessive unicode calculations and text layout

### After
- CPU: ~15-20% during playback
- Rendering 30 FPS only when needed
- Zero FPS when paused (idle)

## What Changed

### Performance Fix
- Frame rate control with 30 FPS cap
- Selective rendering (only when state changes)
- All event handlers properly trigger renders
- Significant reduction in unnecessary TUI updates

### New Features
- Comprehensive performance benchmarking suite
- Real-time CPU/memory monitoring tools
- Performance profiling and analysis tools

### Makefile Targets
```bash
make bench-all       # Run all benchmarks
make bench-watch     # Real-time monitoring
make bench-analyze   # Identify bottlenecks
make bench-results   # Show latest results
```

## Impact

⚡ 50-70% less CPU usage  
🔋 Better battery life  
🌡️ Lower system temperature  
🔇 Quieter fans  
✨ No quality loss or breaking changes
